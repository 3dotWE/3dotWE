About Gust\n
\n
\n \n \n \n GUST Browser\n
\n v1.0\n
\n\n \n
\n \n How it\n works\n
\n Gust is a static web proxy\n browser that runs entirely inside a single HTML file.\n When you navigate to a URL, Gust fetches the target site's HTML,\n CSS, scripts, and assets through Wisp, then rewrites\n all resource URLs and injects them into a sandboxed iframe. Gust can be run inside any HTML editor, iframe, file, or blob. It's ultra lightweight, packed with features, and is near impossible to block for admins.\n \n
\n\n
\n \n \n WISP + libcurl.js\n
\n All network requests are tunnelled through a WISP WebSocket server. Your\n browser opens a WebSocket to the WISP endpoint, which forwards\n TCP/TLS connections on your behalf. The actual HTTP fetching is\n handled by libcurl.js\n — a port of the curl HTTP library compiled to WebAssembly\n — giving full HTTPS, redirect, and cookie support inside the\n browser context.\n \n\n
\n \n Page\n rendering\n
\n Fetched HTML is parsed and rewritten in JavaScript. CSS stylesheets\n and scripts are loaded through the proxy and injected inline or as\n Blob URLs. Images, fonts, and other assets have their URLs rewritten\n to route through the same proxy pipeline. The final document is\n rendered inside a sandboxed iframe\n using a generated Blob URL.\n \n\n
\n \n \n Privacy & limitations\n
\n Because requests flow through the WISP server, your real IP is not\n exposed to the sites you visit. However, the WISP server operator\n may be able to see your traffic.\n Some complex web features (native WebSockets from within pages,\n WebRTC, service workers) cannot be fully proxied and may not work.\n JavaScript inside the iframe is fully sandboxed from the parent\n page.\n \n\n
